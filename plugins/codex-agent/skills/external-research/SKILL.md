# External Research

1. Inspect the repository version, lockfile, local types, and installed documentation first.
2. Prefer a purpose-built connector, API, or CLI for linked resources and private systems.
3. Prefer official documentation and primary repositories for public technical facts.
4. Use the narrowest query that resolves the implementation decision.
5. Record the exact version, date, or capability boundary when relevant.
6. Distinguish verified facts from inference and unresolved uncertainty.
7. Return concise findings with direct citations and the implementation consequence.

For OpenAI and Codex questions, use the official OpenAI documentation source route available in the session.

Read [references/source-policy.md](references/source-policy.md) when sources conflict or authentication blocks access.
