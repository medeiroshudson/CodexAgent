# Evidence Contract

Report:

- `Changed`: concise description of the delivered behavior.
- `Validated`: exact commands or interactions and their outcomes.
- `Not validated`: material platforms, data states, integrations, or flows not exercised.
- `Residual risk`: known limitations or assumptions that remain.
- `Durable context`: omit when there is no qualifying candidate; otherwise provide one concise, evidence-backed proposal separately from completion and request approval before writing it.

Do not say tests passed when a command did not run, timed out, was skipped, or failed for an unexplained reason. Distinguish environment blockers from implementation failures.
