# Task Contract

Each task must contain:

- `id`: stable short identifier.
- `outcome`: observable result.
- `scope`: included and excluded behavior.
- `context`: applicable standards and reference files.
- `dependsOn`: task identifiers that must finish first.
- `parallelSafe`: boolean supported by an overlap analysis.
- `agent`: preferred narrow role.
- `validation`: exact proportional checks.
- `doneWhen`: measurable completion criteria.

Do not create file artifacts for the plan unless the user requests durable planning documents or the workflow needs cross-session handoff.
